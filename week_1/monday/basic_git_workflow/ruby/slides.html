Basic Git Workflow

Setup

first time working on the repo?

  • In terminal, navigate to the directory where you'd like to store the repo

  • Run: git clone repository_url
    - copies a remote github repository to your computer

  • cd into the repo's directory

  • Run: git checkout -b branch_name
    - adds a new branch to the repo, and gives it the name you specified

  • Run: subl .
    - opens the current directory in sublime

Setup

additional work on the repo?

  • In terminal, navigate to the repo's folder

  • Run: git checkout master
    - moves you to the master branch of your repo

  • Run: git pull origin master
    - brings your local master branch up to date with github's

  • Run: git checkout branch_name
    - switches you to the specified branch

  • Run: subl .
    - opens the current directory in sublime

Work

  • In sublime, add code, change code, etc.

  • Run: git add .
    - "stages" all the files in your current working directory

  • Run: git commit -m "a meaningful msg"
    - saves all staged changes, along with a descriptive msg

  • Use git status anytime you want to verify changes and additions.

  • Repeat until finished working. Commit your work often -- very often.

Wrap Up

  • git push origin branch_name
    - pushes your local changes to your online github repo

  • Create a pull request on github
    - this allows folx to review and comment on your changes
